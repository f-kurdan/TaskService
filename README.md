# TaskService

Сервис управления задачами и проектами.

## Назначение

Хранение и управление проектами, задачами, комментариями и тегами. Предоставляет REST API для работы с этими сущностями.

## Зона ответственности

- CRUD для сущностей: `Project`, `TaskItem`, `Comment`, `Tag`, `User`
- Управление статусами задач: `Backlog` → `InProgress` → `Review` → `Done` / `Cancelled`
- Управление приоритетами задач: `Low`, `Medium`, `High`, `Critical`

## Основные сценарии использования

- Создание проекта и задач внутри него
- Назначение исполнителя на задачу
- Смена статуса задачи в процессе работы
- Добавление комментариев к задаче
- Привязка тегов к задачам для категоризации

## Стек

- **.NET 8** — рантайм
- **ASP.NET Core** — HTTP API
- **Entity Framework Core + Npgsql** — доступ к данным
- **PostgreSQL** — база данных

## Архитектура

Слоистая архитектура:

```
TaskService.Api           — HTTP-контроллеры, middleware, фильтры
TaskService.Application   — сервисный слой, DTO, валидация
TaskService.Domain        — сущности, интерфейсы, перечисления
TaskService.Infrastructure — EF Core контекст, миграции, репозитории
```
